# A/B Decisions

## Conservative Rule

Champion A remains selected when the challenger is tied, only slightly better, generated from a different source, or not yet stable across repeated trials.

## walk-cycle

Decision: keep Champion A.

Evidence:

- Champion A already has official mechanical QA pass and an accepted official sample.
- Challenger B v1 passed mechanical QA in 2/2 trials.
- B improves some side-row and diagonal stride readability, but front/back leg-depth remains subtle and trial 002 still has several near-duplicate front/back frames.
- B has not reached Pilot tier because only 2 walk trials were run.

Next hypothesis:

- Keep B's front/back foot-depth wording.
- Add a stricter "front/back shoe depth must alternate every contact beat" line.
- Run a 3-trial Pilot before considering a 5-trial Candidate gate.

## run-cycle

Decision: keep Champion A.

Evidence:

- Champion A already has official mechanical QA pass and a readable official run sample.
- Challenger B v1 passed mechanical QA in 1/1 trial and browser gallery QA.
- B produced a strong side/diagonal run silhouette after additional direction selection, but the run job log records that initial front-three-quarter and side candidates were partly walk-like.
- The need for extra re-generation/selection makes B a useful prompt hypothesis, not a stable official replacement.
- B has not reached Pilot tier because only 1 run trial was run.

Next hypothesis:

- Keep the mandatory crossover and lead-foot-switch wording.
- Add "do not accept walking cadence in side/front-three-quarter rows" to the next B revision.
- Run at least 3 run trials before any replacement discussion.
