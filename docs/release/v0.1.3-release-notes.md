# v0.1.3 Release Notes

Status: published as v0.1.3
Target: Recursive animation QA, review galleries, and calibrated motion gates after v0.1.2

Image Cockpit for Codex Workflows v0.1.3 is a focused animation QA release. It makes the recursive browser-test loop easier to inspect, turns failed sprite candidates into reviewable visual evidence, and adjusts motion gates where human review showed the generated animation was acceptable.

Feature scope is the main branch range from `v0.1.2` through `4ba43b7`, including generated-output rights clarification, animation preset identity contracts, source-selection guard work, recursive browser QA artifacts, failed sprite GIF previews, and the Idle Breathing / Run Cycle / Walk Cycle threshold fixes.

## Highlights

- Added recursive animation QA artifacts under `docs/qa/image-animation-recursive-browser-test-loop/20260630-0150/`.
- Added `index.html` so source images, trial outcomes, reports, screenshots, and summaries can be reviewed from one local page.
- Added `failed-sprite-sheets.html` for failed sprite sheet inspection.
- Added animated GIF previews for failed 5-direction sprite rows, making subtle motion issues easier to judge than static sheets alone.
- Clarified generated-output rights for local Codex/imagegen handoff workflows.
- Improved animation source restoration when returning to Animation Generation from an animation result.
- Preserved direction-split and generic single-image diagnostics so recoverable outbox artifacts are not mislabeled as no-image failures.
- Calibrated Idle Breathing, Run Cycle, and Walk Cycle quality gates after visual review showed the previous motion thresholds were too strict for valid subtle motion.

## Recursive Animation QA

- Six source images were generated and retained as visual-review inputs.
- Twelve animation trials were documented with browser trial JSON, reports, screenshots, and delivery summaries.
- The QA loop recorded zero false-success entries and zero stuck-running entries.
- `artifact-index.md`, `failure-taxonomy.md`, `fix-cycles.md`, `verification.md`, `index.html`, and `failed-sprite-sheets.html` now provide the handoff path for future matrix reviews.

## Motion Gate Calibration

- Idle Breathing now accepts two readable moving directions while preserving hard static-copy guards for near-identical frames.
- Run Cycle standard motion gates were relaxed after the back row showed readable running motion below the previous generic threshold.
- Walk Cycle now uses a dedicated walk motion profile so small-character walk loops are not blocked by generic larger-motion thresholds.
- The old failed trial evidence remains in the QA folder for auditability, while the implementation now treats those reviewed cases as acceptable under their motion-specific profiles.

## Review Galleries

- `index.html` lists generated sources, animation trials, statuses, screenshots, and linked reports.
- `failed-sprite-sheets.html` shows the rejected 5-direction sheet rows and matching animated GIF previews.
- The GIF previews are generated from committed PNG rows and verified for frame count, size, and timing.

## Local-First Boundary

The app still does not call OpenAI APIs directly and does not require an API key. It writes local handoff files for Codex, the user, or another local workflow to process.

No model weights, API keys, tokens, generated handoff folders, or license-unclear sample assets are included in the repository.

## Known Limitations

- Image generation and image editing still depend on the local Codex/imagegen environment available to the user.
- The recursive QA snapshot keeps failed pre-fix trials as historical evidence; not every recorded failed row was regenerated after the threshold change.
- Direct OpenAI API, ComfyUI, AUTOMATIC1111, Replicate, and other provider adapters are still not part of the shipped baseline.
- Advanced game-engine exporters, full background removal, and deeper sprite QC remain future work.
