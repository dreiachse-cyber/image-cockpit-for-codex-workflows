# v0.1.2 Release Notes

Status: published as v0.1.2
Target: Animation delivery reliability, recovery hardening, and long-running job feedback after v0.1.1

Image Cockpit for Codex Workflows v0.1.2 is a reliability-focused release for animation generation. It keeps the project local-first and Codex-oriented while making successful animation delivery more measurable, failed candidates less confusing, and long-running jobs easier to trust.

Feature scope is the main branch range from `v0.1.1` through `91d75e4`, including settings recovery reporting, detached direction-split recovery, animation result quality gates, first-two candidate adoption, animation delivery SLO evidence, and running Codex job progress indicators.

## Highlights

- Added animation delivery reliability tooling and QA evidence based on real browser generation trials.
- Added current-regime animation delivery rollup checks so recent successful behavior can be measured separately from pre-fix failures.
- Added running Codex job progress indicators to the right job panel, compact log cards, and fullscreen log header.
- Added settings recovery and environment report materials so users can copy local diagnostics when reporting problems.
- Improved recovery of detached direction-split outputs when verified direction PNGs and manifests are present.
- Hardened animation result quality gate handling so raw, debug, bronze, contact-sheet, partial, or low-quality outputs do not appear as final successes.
- Adopted first two usable candidates for standard animation tournament AB scoring, publishing only the winning final artifact.
- Extended animation delivery wait behavior to reduce false failures when real Codex/imagegen jobs are slow but still progressing.

## Animation Delivery Reliability

- Added `npm run animation:delivery`, `npm run animation:baseline`, and `npm run animation:rollup`.
- Added browser delivery smoke coverage for history, preview, download panel, and root outbox delivery.
- Added baseline and rollup documentation under `docs/qa/animation-delivery-reliability/`.
- Current-regime real browser delivery evidence reached the documented SLO before release prep.
- The long-timeout real browser sample recorded 4/4 successful deliveries after extending stale handling for slow jobs.

## Quality Gate And Recovery

- Direction-split imports now prefer verified final artifacts and avoid treating partial or stale files as complete results.
- Animation quality failures are quarantined instead of being added to normal history/download flows.
- Local generator artifacts, temporary contact sheets, debug sheets, QA files, and work files are filtered from normal recovery paths.
- Failed hidden tournament candidates keep diagnostic evidence without polluting the user-facing result list.
- Completed jobs that only produced invalid animation artifacts are classified as failures instead of false successes.

## Operator Feedback

- Running job cards now show a coarse progress bar, elapsed time, and phase text.
- The progress indicator intentionally avoids fake exact percentages and never reaches a completed-looking 100% while the job is still running.
- Compact and fullscreen Codex logs show the same running-state feedback.
- Reduced-motion settings disable shimmer animation.

## Local-First Boundary

The app still does not call OpenAI APIs directly and does not require an API key. It writes local handoff files for Codex, the user, or another local workflow to process.

No model weights, API keys, tokens, generated handoff folders, or license-unclear sample assets are included in the repository.

## Known Limitations

- Image generation and image editing still depend on the local Codex/imagegen environment available to the user.
- The separate two-job animation tournament branch is not included in v0.1.2; this release includes the main-merged first-two usable candidate adoption.
- Current animation matrix QA work is still ongoing and is not part of this tag.
- Direct OpenAI API, ComfyUI, AUTOMATIC1111, Replicate, and other provider adapters are still not part of the shipped baseline.
- Advanced game-engine exporters, full background removal, and deeper sprite QC remain future work.
