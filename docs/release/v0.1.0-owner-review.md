# v0.1.0 Owner Review Guide

更新日: 2026-06-23

このファイルは、ご主人がprivate MVPを短時間で確認するための入口です。

## 先に結論を見る

- 目的: 画像生成 / 画像編集 / スプライトシート生成 / スプライトシート編集の4大ワークフローが、ローカルCodex handoff中心で分かりやすく行えるかを確認する
- 境界: アプリ自身はOpenAI APIを直接呼ばず、API keyも不要
- 状態: draft PRのprivate release candidate
- PR: https://github.com/dreiachse-cyber/image-cockpit-for-codex-workflows/pull/1
- 残り: ご主人レビュー、release notes承認、main merge / public化 / tag / GitHub release承認

## 1. 自動確認

Codexが入っているレビュー機では、まず以下を通します。

```powershell
npm run review:local
```

これは以下を順に実行します。

- `npm run verify`
- `npm run ui:smoke`
- `npm run codex:smoke`

期待結果:

- doctor / typecheck / test / build / smoke / release audit が通る
- Guided Start、4大workflow、言語切替、sprite-edit中核controlがheadless browserで通る
- 実Codex CLIのno-image handoff jobが `running -> completed` になり、outboxへMarkdown sidecarを書く

## 2. 手触り確認

自動確認後、アプリを触ります。

```powershell
npm run dev:all
```

確認ポイント:

1. Guided Startの4択が迷わないか
2. `1. 画像生成` でCodex jobを作り、outbox返却を `最新を取り込み` で戻せるか
3. `2. 画像編集` で画像、注釈、編集メモがhandoff jobに入るか
4. `3. スプライトシート生成` で画像を取り込み、grid splitからtimelineへ入れるか
5. `4. スプライトシート編集` で順番、anchor、透明化、PNG / ZIP / GIF / metadata exportを確認できるか
6. 日本語 / English 切り替えで中核ラベルが追従するか
7. 低優先の細かい機能に導線が引っ張られていないか

## 3. 判断ファイル

- 詳細レビュー: `docs/review/mvp-review-report.md`
- 受け入れ証跡: `docs/release/v0.1.0-acceptance-evidence.md`
- 承認ゲート: `docs/release/v0.1.0-owner-decision.md`
- 公開手順: `docs/release/v0.1.0-runbook.md`
- release notes草案: `docs/release/v0.1.0-release-notes.md`

## 4. 承認する場合

以下は個別に明示承認が必要です。

- release notes / changelog承認
- final PR diff承認
- main merge承認
- repository public化承認
- `v0.1.0` tag / GitHub release承認

## 5. 承認しない場合

気になる点をP0 / P1 / P2で返します。

- P0: release不可の重大問題
- P1: 初回公開前に直すべき問題
- P2: 公開後でもよい改善

P0/P1がある場合は、privateのままPRへ追加修正します。
