# v0.1.0 Owner Review Guide

更新日: 2026-06-23

このファイルは、v0.1.0公開時のowner review履歴と、ローカル確認手順を短時間で追うための入口です。

## 先に結論を見る

- 目的: ピクセルアート生成 / 画像編集 / アニメーション生成の主要ワークフローが、ローカルCodex handoff中心で分かりやすく行えるかを確認する
- 境界: アプリ自身はOpenAI APIを直接呼ばず、API keyも不要
- 状態: v0.1.0公開済み
- repo: https://github.com/dreiachse-cyber/image-cockpit-for-codex-workflows
- 残り: 公開後のREADME / GitHub表示 / X告知素材の確認

## 1. 自動確認

Codexが入っているレビュー機では、まず以下を通します。

```powershell
npm run review:local
```

これは以下を順に実行します。

- `npm run verify`
- `npm run ui:smoke`
- `npm run codex:smoke`

期待結果:

- doctor / typecheck / test / build / smoke / release audit が通る
- 3つのworkspace tab、言語切替、prompt examples、animation preset、download flowがheadless browserで通る
- 実Codex CLIのno-image handoff jobが `running -> completed` になり、outboxへMarkdown sidecarを書く

## 2. 手触り確認

自動確認後、アプリを触ります。

```powershell
npm run dev:all
```

確認ポイント:

1. Pixel Art Generation / Image Editing / Animation Generation の3タブが迷わないか
2. Pixel Art GenerationでCodex jobを作り、outbox返却または自動取り込みでhistoryへ戻せるか
3. Image Editingで画像、矩形注釈、編集コメントがhandoff jobに入るか
4. Animation Generationでsource画像、motion preset、spriteContextがhandoff jobに入り、direction GIF / sprite sheet / animation packを書き出せるか
5. 日本語 / English 切り替えで中核ラベルが追従するか
6. 低優先の細かい機能に導線が引っ張られていないか

## 4. 承認する場合

以下は個別に明示承認が必要です。

- README / release docsの公開後文言確認
- GitHub About / Topics / Social preview確認
- PR受付制限の確認
- X投稿文と添付素材の確認

## 5. 承認しない場合

気になる点をP0 / P1 / P2で返します。

- P0: 公開表示を止めるべき重大問題
- P1: X告知前に直すべき問題
- P2: 公開後でもよい改善

P0/P1がある場合は、公開表示または告知素材へ追加修正します。
