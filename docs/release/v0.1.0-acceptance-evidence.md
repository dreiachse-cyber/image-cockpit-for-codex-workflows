## Verification Commands

The release candidate verification path is:

```powershell
npm run doctor
npm run typecheck
npm test
npm run build
npm run smoke
npm run release:audit
```

Expected result: all commands pass. `npm run doctor` may warn that the Codex executable is unavailable in environments where Windows blocks `codex exec`; that is acceptable for private MVP review when manual handoff is being reviewed.

## Remaining Gates

The following are not completed by this document and still require explicit owner decision:

- Owner review of `docs/review/mvp-review-report.md`
- Codex executable verification on an environment where `codex exec` can run without `spawn EPERM`, or an explicit manual-handoff-first release decision
- Approval of `docs/release/v0.1.0-release-notes.md`
- Approval to merge into `main`
- Approval to change repository visibility from private to public
- Approval to create the `v0.1.0` tag and GitHub release

## Do Not Treat As Accepted

- Direct OpenAI API integration from the app
- Required `OPENAI_API_KEY` setup
- ComfyUI, AUTOMATIC1111, Replicate, or other external adapters
- IndexedDB persistence
- Game-engine-specific exports
- Bundled model weights or license-unclear assets
