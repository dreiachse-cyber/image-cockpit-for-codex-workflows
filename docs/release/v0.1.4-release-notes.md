# v0.1.4 Release Notes

Status: experimental, prepared for tag `v0.1.4`
Target: Effect Animation MVP, looping previews, APNG export, and compact result cards after v0.1.3

Image Cockpit for Codex Workflows v0.1.4 is an experimental release for trying the new Effect Animation workflow from the main branch. It keeps the local-first handoff model and adds a focused game VFX path for transparent sprite-sheet effects.

Feature scope is the main branch range from `v0.1.3` through `a3d2a0c`, plus the v0.1.4 release-prep commit. The release includes the Effect Animation MVP, APNG download support, GIF preview loop handling, compact Effect result cards, and the Vite watcher hotfix used while validating the workflow.

## Highlights

- Added `Effect Animation` as a workflow beside Pixel Art Generation, Image Editing, and Animation Generation.
- Added five MVP categories: Slash Arc, Hit Spark, Magic Cast, Projectile, and Impact.
- Added controls for effect type, style, palette, frame count, frame size, sheet layout, loop mode, anchor, blend mode, and preview background.
- Added effect result previews for the transparent sheet, looping GIF preview, frame timeline, quality rank, and download readiness.
- Added downloads for Effect GIF, Effect APNG, Animated APNG-style preview assets, Sheet PNG, Frames ZIP, Metadata JSON, and Effect Pack ZIP.
- Reduced the Results list to compact Effect result cards so long effect sheet filenames and metadata do not break the layout.
- Kept effect results out of numbered Image Editing region flows because they are final animation export assets.

## Effect Animation

Effect jobs use `workflowMode: "effect-animation"` and send effect metadata to the local Codex handoff payload. The runner contract asks for a transparent sheet with readable frame-to-frame motion, no baked checkerboard, no text, no frame numbers, no gutters, and a metadata/GIF sidecar when feasible.

The MVP categories are tuned for small game VFX:

- Slash Arc for crescent slash attack effects.
- Hit Spark for burst and metal contact effects.
- Magic Cast for magic circle and casting effects.
- Projectile for energy bolt effects.
- Impact for small explosion and hit impact effects.

Quality checks cover sheet layout, frame count, frame size, alpha transparency, checkerboard or matte risk, duplicate/static frames, cell bounds, and failed/debug artifact exclusion.

## Export And Preview

- GIF preview is expected to loop in the result preview.
- Effect APNG export is available from the shared download modal.
- Effect Pack ZIP includes `sheet.png`, `effect.json`, `preview.gif`, `preview.apng`, and frame PNGs.
- Animated APNG validation checks the APNG `acTL` chunk, frame count, and loop count.
- Compact Effect result cards show a short category line and rank/frame/size summary instead of long filenames.

## QA Evidence

Primary QA record: `docs/qa/effect-animation-mvp.md`

The QA pass covers deterministic browser smoke trials and real Codex/imagegen import trials for Slash Arc, Hit Spark, Magic Cast, Projectile, and Impact. Each imported result rendered a transparent sheet preview, GIF preview, frame timeline, quality rank, and download panel.

## Local-First Boundary

The app still does not call OpenAI APIs directly and does not require an API key. It writes local handoff files for Codex, the user, or another local workflow to process.

No model weights, API keys, tokens, generated handoff folders, or license-unclear sample assets are included in the repository.

## Known Limitations

- This is an experimental Effect Animation MVP tag, not a final engine-export release.
- Character overlay preview is not included.
- Godot, Unity, and Phaser exporters are not included beyond portable metadata and effect-pack output.
- Real image-model outputs can still vary by environment, so retakes may improve individual art direction.
